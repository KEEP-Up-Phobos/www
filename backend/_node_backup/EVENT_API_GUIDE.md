# Event API Integration Guide

## Overview

KEEP-Up now has a **properly structured API client** that handles all major event platforms with their correct data formats. The system was generated by analyzing official API documentation.


- S-Token header authentication
- Endpoint: `https://api.sympla.com.br/public/v3/events`

#### **Songkick**
- API key in params
- Endpoint: `https://api.songkick.com/api/3.0/events.json`

- Returns `resultsPage.results.event` array

## How to Use

### Step 1: Get API Keys

Sign up for API keys at these URLs (all free tiers available):

1. **Eventbrite**: https://www.eventbrite.com/platform/api
   - Create developer account
   - Create private token
   - Add to `.env` as `EVENTBRITE_API_KEY`

2. **Ticketmaster**: https://developer-acct.ticketmaster.com/user/register
   - Register for free account
   - Get API key from dashboard
   - Add to `.env` as `TICKETMASTER_API_KEY`

3. **Sympla**: https://www.sympla.com.br/
   - Contact them for API access (Brazil-focused)
   - Add to `.env` as `SYMPLA_API_KEY`

4. **Songkick**: https://www.songkick.com/developer
   - Request API key
   - Add to `.env` as `SONGKICK_API_KEY`

### Step 2: Add Keys to .env

Edit `/var/www/KEEP-Up/backend/.env`:

```bash
# Event API Keys
EVENTBRITE_API_KEY=your_actual_key_here
TICKETMASTER_API_KEY=your_actual_key_here
SYMPLA_API_KEY=your_actual_key_here
SONGKICK_API_KEY=your_actual_key_here
```

### Step 3: Run the Fetcher

```bash
# Test the API client
sudo docker exec keepup_node node /app/backend/demo-api-fetch.js

# Fetch Porto Alegre events
sudo docker exec keepup_node node /app/backend/demo-api-fetch.js
```

## Integration with Existing Fetcher

To integrate into `fetcher.js`, add:

```javascript
const EventAPIClient = require('./api-client-generated');

class EventFetcher {
    constructor() {
        this.apiClient = new EventAPIClient();
    }

    async fetchPortoAlegreEvents() {
        const location = 'Porto Alegre, Brazil';
        const apis = ['eventbrite', 'ticketmaster', 'sympla', 'songkick'];
        
        for (const api of apis) {
            const result = await this.apiClient.searchEventsByLocation(
                api, 
                location, 
                { limit: 100, countryCode: 'BR' }
            );
            
            if (result.events.length > 0) {
                await this.apiClient.saveToDatabase(result.events, this.db);
            }
        }
    }
}
```

## Troubleshooting

### "401 Unauthorized"
- Missing or invalid API key
- Check that key is in `.env` and matches the required format

### "404 Not Found"
- Incorrect endpoint URL
- API may have changed - run `api-doc-scraper.js` to regenerate

### "403 Forbidden"
- Rate limit exceeded
- IP blocked - wait or use proxy
- API key not approved yet

### "No events found"
- No events in that location
- Try different city or broader search radius
- Check if API key has correct permissions

## .NET Event-Serpent Microservice (Jan 22, 2026)

### Overview
A new .NET 8.0 WebAPI service has been implemented for high-performance event fetching:

- **Location:** `/dotnet/event-serpent/`
- **Technology:** ASP.NET Core Minimal API
- **Container:** Dockerized with multi-stage build
- **Integration:** Called first by serpents-service orchestration layer

### Implementation Details
- **Program.cs:** Minimal API with Ticketmaster HTTP client integration
- **Fallback:** Mock events when API unavailable
- **Response Format:** 
  ```json
  {
    "events": [...],
    "total": 2,
    "time_seconds": 0.5,
    "errors": 0
  }
  ```
- **Endpoint:** `GET /api/serpents`
- **Port:** 5000 (127.0.0.1) / 80 (Docker)

### Testing
```bash
# Build and run
docker-compose build dotnet-eventserpent
docker-compose up -d dotnet-eventserpent

# Test API
curl http://127.0.0.1:5000/api/serpents
```

### Orchestration
The Node.js serpents-service (`/backend/serpents-service/`) calls this .NET service first, then falls back to legacy Node.js fetchers if unavailable.
