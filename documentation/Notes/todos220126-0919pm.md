## ✅ Todo #6: Implement Bcrypt for New Registrations

### Problem Description
New user registrations use MD5:salt hashing, but Joomla 6.0 uses bcrypt ($2y$).

### Current State
- `joomla_password.js` verifies both bcrypt and MD5:salt ✅
- New registrations create MD5:salt hashes ❌
- Should use bcrypt for security

### Files to Modify
```
/backend/joomla_password.js
  - Update hashPassword() to use bcrypt
  - Keep verify() supporting both formats

/backend/main_server.js
  - Registration endpoint uses new bcrypt hash
```

### Implementation
```javascript
// joomla_password.js - new hashPassword function
async function hashPassword(password) {
  const bcrypt = require('bcrypt');
  const saltRounds = 10;
  // Use $2y$ prefix for Joomla compatibility
  const hash = await bcrypt.hash(password, saltRounds);
  // Convert $2b$ to $2y$ for Joomla
  return hash.replace('$2b$', '$2y$');
}
```

## ✅ Todo #7: Create Single requireAuth Middleware

### Problem Description
Token validation is duplicated in 4+ places in main_server.js.

### Current Locations
1. `app.get('/api/auth/check')` — line 584
2. `requireJoomlaSession()` middleware — line 500
3. `requireAdminToken()` middleware — line 1000
4. `auth_sync_client.js` — line 60

### New Unified Middleware
```
/backend/middleware/auth.middleware.js
```

### Implementation Plan
```javascript
// auth.middleware.js
const { verifySession } = require('../unified-auth');

function requireAuth(options = {}) {
  return async (req, res, next) => {
    // 1. Check Authorization header
    const authHeader = req.headers.authorization;
    // 2. Check cookie
    const cookieToken = req.cookies?.KEEPUP_SESSION;
    // 3. Check query param (legacy)
    const queryToken = req.query.token;
    
    const token = authHeader?.replace('Bearer ', '') || cookieToken || queryToken;
    
    if (!token) {
      return res.status(401).json({ error: 'No token provided' });
    }
    
    const session = await verifySession(token);
    if (!session) {
      return res.status(401).json({ error: 'Invalid session' });
    }
    
    req.user = session.user;
    req.session = session;
    next();
  };
}

module.exports = { requireAuth };
```
